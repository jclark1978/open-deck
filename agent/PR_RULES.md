# Pull Request Rules

## Purpose

Ensure all changes are reviewed, understandable, and safe.

---

## PR Requirements

Each PR must include:

### Summary
A short description of what changed

### Changes
- A bullet list of the concrete changes

### Reason
Why the change was needed

---

## Size Guidelines

- Prefer small PRs
- Avoid massive changes unless necessary

---

## Agent Behavior

Before opening a PR:
- Review code for clarity
- Remove unnecessary changes
- Ensure consistency with standards
- Make sure the PR is scoped to one logical change
